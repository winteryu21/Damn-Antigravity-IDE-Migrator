# 기여 안내 (Contributing Guidelines)

**Damn Antigravity IDE Migrator** 프로젝트에 기여해 주셔서 감사합니다! 
이 프로젝트는 기존 Antigravity 에디터의 중요한 설정과 소중한 대화 기록들을 새로운 Antigravity IDE로 안전하고 완벽하게 이식하는 것을 목표로 합니다. 

어떤 작은 도움(오타 수정, 버그 리포트, 코드 제안 등)이라도 언제나 환영합니다.

## 1. 이슈(Issue) 작성 규칙

문제를 제보하거나 새로운 아이디어를 제안할 때는 원활한 소통과 빠른 해결을 위해 아래 가이드라인을 지켜주세요.

### 버그 리포트 (Bug Report)
버그를 제보하실 때는 문제 해결을 위해 아래 내용을 꼭 적어주세요:
1. **실행 환경**: OS 버전 (예: Windows 10/11) 및 기존 Antigravity 버전
2. **재현 방법**: 어떤 순서로 버그가 발생했는지 단계별로 적어주세요.
3. **로그 정보**: 프로젝트 루트 디렉터리에 기록된 `migration.log` 파일 내용이나 터미널 오류 메세지 전문을 첨부해 주세요.

### 기능 제안 (Feature Request)
새로운 기능이나 스크립트 옵션 추가를 제안하고 싶으실 때는 다음을 적어주세요:
1. **제안 배경**: 현재 어떤 점에서 사용이 불편하고 왜 이 기능이 필요한지 설명해 주세요.
2. **희망하는 개선점**: 제안하는 기능의 작동 방식이나 기대 효과를 설명해 주세요.

---

## 2. 코드 기여 과정 (Pull Request)

직접 코드를 수정하여 기능을 개선하거나 버그를 해결하고 싶으시다면 다음 절차를 따릅니다:
1. 저장소를 **Fork**한 뒤, 새로운 작업 브랜치를 생성해 작업합니다.
2. 작업 완료 후 로컬에서 단위 테스트를 수행하여 모든 테스트가 통과하는지 확인합니다.
3. 이 저장소로 **Pull Request(PR)**를 발송하며, 본문에 반영된 변경사항 및 수정 이유를 간략히 설명해 주세요.

## 3. 개발 환경 설정 및 테스트

이 도구는 의존성을 최소화하기 위해 **파이썬 표준 라이브러리**만 사용합니다. 추가 라이브러리 설치가 필요 없습니다.

* **파이썬 요구 사양**: Python 3.8 이상
* **단위 테스트 실행**:
  코드 변경 후에는 반드시 아래 명령어를 통해 기존 기능이 깨지지 않는지 검증해야 합니다.
  ```bash
  python -m unittest tests/test_migration.py
  ```

## 4. 코드 작성 규칙

* **표준 라이브러리 지향**: 외부 의존성(pip install)을 추가하지 않고 파이썬 내장 라이브러리만 사용합니다.
* **안전제일 설계**: 파일이나 데이터베이스를 변경하기 전에 반드시 자동 백업이 작동하고 롤백이 가능하도록 로직을 설계해야 합니다.
* **코드 스타일**: PEP 8 표준을 준수하고 직관적인 변수명/함수명을 사용해 주세요.
